# База знаний

Сюда складывай **устойчивые знания** (заметки, решения, сниппеты). Это то, что агент должен "помнить".

## Рекомендованная структура

- `general/` — общие принципы, стиль, подход к проектированию
- `go/` — Go: ошибки, тестирование, структуры пакетов
- `kmm/` — Kotlin Multiplatform Mobile: архитектура, shared module, networking, DI
- `flutter/` — Flutter: state management, навигация, проектная структура

## Как писать заметки (просто)

- Пиши **коротко**.
- Добавляй **пример кода** или “если X → делай Y”.
- Если есть спорный момент — фиксируй “я выбираю так-то потому что …”.
